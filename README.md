# silly.ehis.gay

It literally provides any image/GIFs, for silly reasons, that's it :3

You can simply fork the repo, do `npm install` to get the packages and run it with `npm run dev` (or `npm run start`)

## Adding images

1. Simply do a [Fork](https://github.com/huhuhuhuheh/silly.ehis.gay/fork) to your account
2. Grab your fork's source code via the code --> Download ZIP or via git (`git clone <repo>`)
3. Switch to new branch (`git switch -c <name>` that's unique to your other submissions)
4. Add image at `/public` from the ones that are supported
5. Commit it
6. Push to the branch repo of your fork
7. Make an Pull Request after
8. That's it

### Rules to image submissions
These are easy, and straightforward to follow as long as you know what you're doing:

1. As long as it doesn't contain any bad thing (NSFW or per-so), it gets approved *(must be sfw!)*
   1. If your Pull Request needs to change things inside of the route.ts *(Where it serves the silly things)* or from the next.js project, you must exactly say why *(An exception is made for supporting a file format alongside your Pull Request)*
2. If you know the author of the image, say in the Pull Request alongside its source
     1. If the author provides a watermark, that can also be accepted
     2. If the author requires permissions before using the art somewhere outside of where it is posted at, include in the Pull Request as validation of proof *(Censoring at least information that seems sensitive that is)*
3. Do not save or submit PRs that contain and be named `image` since it can be overwritten, just name it whatever you want before saving it and submitting!!!
## File images supported

- .gif
